# SureVision AI - Technical Project Documentation

## 1. Project Overview & Problem Statement

SureVision AI is an enterprise decision intelligence platform. Modern executives often make high-stakes operational changes (e.g., swapping a logistics vendor, automating an AP workflow) based on static spreadsheets and biased human proposals. This leads to hidden compliance risks, SLA leakage, and unexpected cost overruns.

SureVision AI solves this by intaking enterprise decision proposals and simulating outcomes using a custom LLM orchestration layer (Gemini). It predicts hidden risks, models standard ROI and Trust scores, calculates the **Cost of Inaction (COI)**, and generates board-ready phased rollout plans.

## 2. Architecture & Tech Stack

The platform is designed as a decoupled full-stack Monorepo with four main components:

### Frontend
- **Framework:** Next.js 15 (App Router)
- **Language:** TypeScript
- **Styling:** Tailwind CSS, Framer Motion (for premium micro-animations)
- **State Management:** Zustand (auth-store, dissent-store)
- **Component System:** Custom UI library (Lucide icons, Radix primitives)
- **Charts:** Recharts

### Backend
- **Framework:** Node.js (Express.js)
- **Language:** TypeScript
- **Database:** MongoDB (Mongoose ORM)
- **Authentication:** JWT (access + refresh tokens), bcryptjs (12 rounds)
- **Services:** Nodemailer (SMTP), PDFKit (PDF generation), CSV-Stringify
- **Validation:** Zod

### AI Engine
- **Framework:** Python (FastAPI)
- **Language:** Python
- **Primary Core:** Google Gemini LLM API (Gemini 1.5/2.5 Flash)
- **Integration:** Evaluates deterministic payload algorithms against predictive NLP. Includes a dynamic trust re-evaluation engine for stakeholder dissent.

### Shared
- **Type Definitions:** Shared Zod schemas between frontend/backend
- **API Types:** Shared TypeScript interfaces

## 3. Core Data Flow & User Journey

### Authentication Flow
1. User registers with name, email, password, company, phone, socials
2. System sends verification email (token-based)
3. User must verify email before login (security measure)
4. Login returns JWT access token (short-lived) + refresh token (long-lived)
5. Token refresh maintains session persistence
6. Password reset via secure email token
7. All auth events logged to audit trail

### Decision Analysis Flow
1. **Input:** User submits decision via form with:
   - Title, description
   - Department (Finance, Operations, HR, IT, Sales, Legal)
   - Industry (Banking, Insurance, Healthcare, Logistics, Manufacturing)
   - Budget impact (currency)
   - Time horizon (30, 90, 365 days)
   - Stakeholders affected
   - Current pain point
   - Urgency level (low/medium/high/critical)
   - Compliance sensitivity (low/medium/high)

2. **AI Processing:** Backend sends to Gemini via AI Engine, returns structured analysis

3. **Output:** Results displayed on dashboard with:
   - Recommendation (Approve/Revise/Reject/Pilot)
   - 5 scoring dimensions (Risk, Trust, Compliance, ROI, Human Impact)
   - Executive summary
   - Hidden risks (3-5)
   - Silent patterns (3-5)
   - Safer alternative
   - Phased rollout plan (3-5 steps)
   - Boardroom debate perspectives

### Stakeholder Dissent Flow
1. Stakeholder submits objection with category, severity, rationale
2. AI re-evaluates trust score based on severity:
   - Minor: -1 to -5 points
   - Major: -10 to -15 points
   - Blocking: -20 to -40 points
3. Trust score animates dynamically on dashboard
4. Admin can resolve/dismiss with note
5. PDF export shows warning if blocking objections exist

### Export Flow
1. **PDF:** Generates board-ready document with signature blocks
2. **CSV:** Single-row spreadsheet for data analysis
3. **Email:** HTML email with dark-mode template

## 6. Cost of Inaction (COI) Calculation

Formula:
```
dailyBleed = Math.round((budgetImpact * (riskScore / 100)) / 30)
monthlyLeakage = dailyBleed * 30
```

Example: $1M budget, 60 risk score
- Daily: $20,000/day
- Monthly: $600,000/month

## 11. Native Desktop Application (Electron)

To support secure, enterprise-grade deployment without browser limitations, SureVision AI features a native Windows desktop client built using Electron 32. This application bypasses standard browser CORS policies and local file execution restrictions.

### Desktop Tech Stack & Architecture
- **Core Engine:** Electron v32.0.0 (Chromium v128 + Node.js v22).
- **Frontend Logic:** High-performance Vanilla JavaScript paired with HTML5/CSS3. Avoids heavy frameworks (React/Next) for the renderer to achieve near-instant startup times and zero-latency UI updates.
- **IPC Architecture:** Uses `contextBridge.exposeInMainWorld` in `preload.js` to securely expose native Node APIs (like `window.electronAPI.window.minimize()`) to the isolated renderer process.

### Deep Native Integrations
1. **Seamless Local Backend Spawning:**
   - The Electron `main.ts` process uses `child_process.spawn("npm", ["run", "dev"])` to silently boot the Express backend in the background on port 5000 upon launch.
   - It intercepts `stdout` and `stderr` from the backend and pipes it to `electron-log` for comprehensive system debugging.
   - During shutdown, the app securely captures the exact PID of the spawned backend and executes `taskkill /pid [PID] /T /F` to prevent orphaned background processes and `EADDRINUSE` port conflicts on the next run.
2. **System Tray & Window Management:**
   - Implements a frameless window design (`frame: false`).
   - Uses `TrayManager` to minimize the application to the Windows System Tray, allowing the Copilot to run passively in the background.

## 12. Voice Assistant & Wake Word Engine

The desktop app includes a fully functional, hands-free Gemini Voice Assistant built directly into the UI layer (`app.js`).

### Advanced Voice Engine Mechanics
- **Engine Selection:** Uses the native `webkitSpeechRecognition` API (Chromium's internal HTML5 Speech Engine).
- **Security Bypass:** Electron strictly disables microphone access for isolated renderer processes by default. We implemented a custom `win.webContents.session.setPermissionRequestHandler` in `main.ts` that explicitly intercepts and automatically grants the `media` permission, allowing the Speech Engine to function without prompting the user.
- **Dual-Mode System:**
  - **Passive Mode (Wake Word):** The engine is instantiated with `recognition.continuous = true` and `recognition.interimResults = true`. It runs an infinite background loop listening specifically for the phrases: `"Hey Gemini"`, `"Hey Copilot"`, or `"Hey SureVision"`.
  - **Auto-Restart Resiliency:** Because Chromium automatically halts speech recognition upon detecting prolonged silence or network blips, the `recognition.onend` event is hooked to instantly trigger a `setTimeout(() => recognition.start(), 500)` loop. This guarantees the Wake Word engine never sleeps as long as the user is authenticated.
  - **Active Mode:** Upon detecting the wake word, the system sets `wakeWordMode = false`, restarts the recognition engine to capture the exact user command, and slides the Glassmorphism Assistant Panel into view.

### Intent Parsing & Gemini Fallback
- **Local Intent Router:** A regex-based natural language parser immediately scans the incoming transcript. Simple commands like `/(dashboard|home)/i` or `/(sign out|logout)/i` are executed locally in $<10ms without requiring a backend API call.
- **Smart Gemini Routing:** If a complex, unstructured business scenario is dictated (e.g., *"What is the cost of inaction if we close our London branch?"*), the parser recognizes it as a non-navigation intent. It automatically switches the UI to the **Manual Input** screen, pastes the transcribed text into the Context Box, and prepares the exact prompt for the Gemini AI engine.

## 13. Workforce Intelligence Module

The Workforce AI module provides automated, scaleable employee performance evaluations.

### Deep Technical Workflow
1. **Bulk CSV Data Ingestion:**
   - The desktop app provides a custom Native Dropzone (`#fileDropzone`).
   - Uses the browser `FileReader` API to parse local files and packages them into a `FormData` object.
   - Pushes the payload to the backend `/api/employees/upload` via a secure, token-authenticated `fetch` request.
2. **Data Pipeline:**
   - The Node.js backend uses `multer` to intercept the multipart form data in memory.
   - Converts the CSV buffer into an array of JSON objects, strips whitespaces, validates required headers (`Name`, `Role`, `Quality Score`, `Tasks Completed`), and bulk-inserts them into the MongoDB `Employee` collection.
3. **AI Performance Evaluation Trigger:**
   - When an admin clicks "Run AI Evaluation", the frontend hits `/api/employees/:id/evaluate`.
   - The backend retrieves the employee's historical data and metrics, constructing a structured prompt.
   - The payload is forwarded to the Python AI Engine (`/evaluate-workforce`).
   - The Gemini 1.5 Flash model parses the metrics, identifies trends (e.g., dropping quality scores but rising task counts), and generates a structured JSON review detailing Strengths, Weaknesses, and an algorithmic ROI score.

## 14. Live Production Infrastructure

The platform uses a decoupled microservices architecture deployed across Vercel and Render.

### Frontend: Vercel Hosting
- **URL:** `https://surevision-ai-gngj.vercel.app`
- **Framework:** Next.js 15 (App Router).
- **Configuration:** Employs aggressive static generation (`cache: "force-cache"`) where possible.
- **Environment Targeting:** Uses `NEXT_PUBLIC_API_URL=https://surevision-ai-e9vy.onrender.com/api` to dynamically route all `api.ts` `fetch` requests to the remote Render backend, ensuring CORS compliance and secure cross-origin resource sharing.

### Backend: Render Web Service (Node.js)
- **URL:** `https://surevision-ai-e9vy.onrender.com/api`
- **Architecture:** Express.js REST API.
- **Database:** Connects securely to MongoDB Atlas via the `MONGO_URI` environment variable.
- **Security:** Protected by `helmet`, `express-rate-limit` (180 req/min), and `cors` configured exclusively for the Vercel origin.
- **AI Proxying:** All AI-related endpoints act as an API Gateway, securely proxying requests to the Python AI Engine using `axios`, shielding the Python service from direct internet access.

### AI Engine: Render Web Service (Python/FastAPI via Docker)
- **URL:** `https://surevision-ai.onrender.com`
- **Containerization:** Deployed using a multi-stage Dockerfile to minimize image size and compile native Python extensions (`pydantic-core`).
- **Dependencies:** explicitly locked via `requirements.txt` (FastAPI, Uvicorn, Google-GenAI, Pandas).
- **LLM Integration:** Directly connects to the Google AI Studio via `GEMINI_API_KEY`, utilizing the state-of-the-art `gemini-1.5-flash` model for high-throughput, low-latency reasoning.
